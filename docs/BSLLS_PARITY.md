# Согласование с BSLLS (BSL Language Server)

Канонические коды `BSLxxx` и имена правил BSLLS сопоставлены в коде (`_BSLLS_NAME_TO_CODE`, `RULE_METADATA`) и в [bsl_rules_matrix.md](bsl_rules_matrix.md). Рантайм **не** вызывает Java-анализатор BSLLS; сверка с JAR — опциональная задача у разработчика (см. [BSLLS_BASELINE.md](BSLLS_BASELINE.md)).

**CST:** BSL001 использует узлы ERROR/MISSING дерева tree-sitter; часть правил — обход CST в `diagnostics_cst.py` при чистом парсе. Подробности: [cst_policy.md](cst_policy.md).

## Намеренные отличия onec-hbk-bsl

Дополнительные правила или иной порог без противоречия BSLLS отражаются в матрице. Примеры кодов, дублирующих семантику BSLLS и отключённых по умолчанию: **BSL080**, **BSL092**, **BSL107** (см. `DEFAULT_DISABLED` в `diagnostics.py`).

**BSL256 / BSL208:** эвристика омоглифов в `diagnostics.py` может расходиться с BSLLS Typo.

**BSL065 / BSL153 в модуле формы** (`path_is_likely_form_module_bsl`): поведение согласовано с типичным профилем BSLLS для форм.

**BSL036 / BSL153:** сложность условия — от `Если`/`ИначеЕсли` до `Тогда`; BSL153 подавляется при срабатывании BSL036 на той же конструкции.

**BSL011 / BSL019:** когнитивная сложность учитывает логические операторы в теле метода.

**BSL046 / BSL199:** при включённых обоих — только **BSL199** на **КонецЕсли** для цепочки без `Иначе`.

**BSL055:** не более одной пустой строки подряд (`MAX_BLANK_LINES=1`).

Правила клиент/сервер: [CLIENT_SERVER_DIAGNOSTICS.md](CLIENT_SERVER_DIAGNOSTICS.md). Крупные расхождения чаще всего у **BSL256**; намеренные отличия фиксируйте в матрице правил.

## Справка 1С

При расхождении кодов: проверьте строку в исходнике, смысл правила по документации платформы; намеренные отличия — в [bsl_rules_matrix.md](bsl_rules_matrix.md).
